Define theme

In our main.scss we call @include mat.core(), to include the base styles for Angular Material core. We include this here, in our styling entry point, so that you only have to load a single css file for Angular Material in your app.

In the custom-themes folder we have two theme files, light-theme.scss and dark-theme.scss, where we set our primary, accent & warn colors. The theme-toggle.component.ts initially loads the theme through initializeThemeFromPreferences(), where we check whether there's an explicit preference in localStorage, otherwise initialize based on the prefers-color-scheme media query.

File organization

The .scss files are organized in the following sub-directories:

  • 0-base: General styles affecting the whole application.
  • 1-layouts: Styles for areas/components related to the layout of the application, such as header, footer, content-layout, etc.
  • 2-modules: Styles for specialized components (such as buttons, code, labels, etc.) and specific pages (such as API list page, "Features" page, etc.).
  • custom-themes: Entry-points for the different themes available in the application (in this case light and dark).

There are also some top-level files in src/styles/:

  • _app-theme.scss: Defines a theme() Sass mixin for creating an application theme.
  • _constants.scss: Defines several constants to be used throughout the styles.
  • _mixins.scss: Defines Sass mixins to be used throughout the styles.
  • main.scss: Styles entry-point.
